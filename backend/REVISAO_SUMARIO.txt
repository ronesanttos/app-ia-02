production no Render. O código estava bem estruturado, mas faltavam arquivos 
críticos para deployment.

PROBLEMAS ENCONTRADOS (7):
  1. ❌ Procfile não existia → Render não sabia como executar a app
  2. ❌ REDIS_URL não estava em .env.example → Confusão em produção
  3. ❌ Dockerfile configurado para Celery (erro!) → Precisa web + worker
  4. ❌ Settings.py sem fallback para REDIS_URL → Erro se variável vazia
  5. ❌ Database hardcoded SQLite → Filesystem efêmero do Render quebra
  6. ❌ Celery.py com configurações hardcoded → Duplicação com settings.py
  7. ❌ Documentação de deploy ausente → Usuário perdido no processo

SEGURANÇA - IMPORTANTE!
───────────────────────────────────────────────────────────────────────────────

⚠️  NÃO USE os valores padrão de .env.example em PRODUÇÃO:

1. DJANGO_SECRET_KEY
   Gere uma nova:
   $ python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

2. DJANGO_DEBUG
   Em produção SEMPRE: DJANGO_DEBUG=false

3. SECURE_SSL_REDIRECT
   Em produção SEMPRE: SECURE_SSL_REDIRECT=true

4. CORS_ALLOWED_ORIGINS
   Configure com SEUS domínios, não deixe em branco

5. Nunca commit .env (use apenas .env.example)
   .gitignore já está configurado ✅
